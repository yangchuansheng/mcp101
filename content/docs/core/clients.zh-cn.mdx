## 客户端详情

### Claude桌面应用
Claude桌面应用全面支持MCP，可深度集成本地工具和数据源。

**主要特点：**
- 完全支持资源功能，可附加本地文件和数据
- 支持提示模板
- 工具集成，可执行命令和脚本
- 本地服务器连接，增强隐私和安全性

> ⓘ 注意：Claude.ai网页版目前不支持MCP。MCP功能仅在桌面应用中可用。

### Claude Code
Claude Code是Anthropic推出的交互式智能编码助手，通过自然语言命令帮助您提升编码效率。它支持MCP的提示模板和工具集成，同时也能作为MCP服务器与其他客户端集成。

**主要特点：**
- 支持MCP服务器的工具和提示模板
- 通过MCP服务器提供自身工具，方便与其他MCP客户端集成

### 5ire
[5ire](https://github.com/nanbingxyz/5ire)是一款开源跨平台桌面AI助手，支持通过MCP服务器使用各种工具。

**主要特点：**
- 内置MCP服务器可快速开启和关闭
- 用户可通过修改配置文件添加更多服务器
- 开源且易用，适合初学者
- 未来将持续优化MCP支持

### BeeAI Framework
[BeeAI Framework](https://i-am-bee.github.io/beeai-framework)是一个开源框架，用于构建、部署和大规模运行智能代理工作流。该框架包含**MCP工具**功能，简化了MCP服务器与智能代理工作流的集成过程。

**主要特点：**
- 将MCP工具无缝整合到智能代理工作流中
- 快速实例化来自已连接MCP客户端的框架原生工具
- 计划未来支持更多智能代理MCP功能

**了解更多：**
- [在智能代理工作流中使用MCP工具的示例](https://i-am-bee.github.io/beeai-framework/#/typescript/tools?id=using-the-mcptool-class)

### Cline
[Cline](https://github.com/cline/cline)是VS Code中的智能编码助手，可以编辑文件、运行命令、使用浏览器等，每一步操作都会征求您的同意。

**主要特点：**
- 通过自然语言创建和添加工具（如"添加一个网络搜索工具"）
- 通过`~/Documents/Cline/MCP`目录与他人共享Cline创建的自定义MCP服务器
- 显示已配置的MCP服务器及其工具、资源和错误日志

### Continue
[Continue](https://github.com/continuedev/continue)是一个开源AI代码助手，内置完整支持所有MCP功能。

**主要特点：**
- 输入"@"可引用MCP资源
- 提示模板以斜杠命令形式呈现
- 在对话中直接使用内置和MCP工具
- 支持VS Code和JetBrains IDE，兼容各种大语言模型

### Cursor
[Cursor](https://docs.cursor.com/advanced/model-context-protocol)是一款AI驱动的代码编辑器。

**主要特点：**
- 在Cursor Composer中支持MCP工具
- 同时支持标准输入输出（STDIO）和服务器发送事件（SSE）连接方式

### Emacs Mcp
[Emacs Mcp](https://github.com/lizqwerscott/mcp.el)是一个专为与MCP服务器对接设计的Emacs客户端，实现无缝连接和交互。它为[gptel](https://github.com/karthink/gptel)和[llm](https://github.com/ahyatt/llm)等AI插件提供MCP工具调用支持，遵循Emacs的标准工具调用格式，增强了Emacs生态系统中AI工具的功能。

**主要特点：**
- 为Emacs提供完整的MCP工具支持

### fast-agent
[fast-agent](https://github.com/evalstate/fast-agent)是一个Python智能代理框架，通过简洁的声明式API支持创建智能代理和工作流，完全兼容Anthropic和OpenAI模型的多模态功能。

**主要特点：**
- 基于MCP原生类型支持PDF和图像处理
- 提供交互式前端用于开发和诊断智能代理应用，包括直通和回放模拟器
- 内置支持"构建高效智能代理"工作流
- 可将智能代理部署为MCP服务器

### Genkit
[Genkit](https://github.com/firebase/genkit)是一个跨语言SDK，用于构建和集成生成式AI功能到应用程序中。[genkitx-mcp](https://github.com/firebase/genkit/tree/main/js/plugins/mcp)插件使Genkit既能作为客户端使用MCP服务器，也能从Genkit工具和提示模板创建MCP服务器。

**主要特点：**
- 客户端支持工具和提示模板（部分支持资源）
- 在Genkit的开发者UI游乐场中提供丰富的发现功能
- 与Genkit现有工具和提示模板无缝互操作
- 适用于各大厂商提供的多种生成式AI模型

### GenAIScript
使用[GenAIScript](https://microsoft.github.io/genaiscript/)（JavaScript编写）以编程方式组装大语言模型的提示。在JavaScript中编排大语言模型、工具和数据。

**主要功能：**
- 处理提示的JavaScript工具箱
- 使得操作更简单高效的抽象层
- 与Visual Studio Code无缝集成

### Goose
[Goose](https://github.com/block/goose)是一个开源AI代理，通过自动化编码任务增强您的软件开发能力。

**主要功能：**
- 通过工具向Goose公开MCP功能
- 可以直接通过[扩展目录](https://block.github.io/goose/v1/extensions/)、命令行或UI安装MCP
- Goose允许您通过[构建自己的MCP服务器](https://block.github.io/goose/docs/tutorials/custom-extensions)扩展其功能
- 包含用于开发、网页抓取、自动化、记忆以及与JetBrains和Google Drive集成的内置工具

### LibreChat
[LibreChat](https://github.com/danny-avila/LibreChat)是一个开源、可定制的AI聊天UI，支持多种AI提供商，现在包括MCP集成。

**主要功能：**
- 通过MCP服务器扩展当前工具生态系统，包括[代码解释器](https://www.librechat.ai/docs/features/code_interpreter)和图像生成工具
- 使用来自顶级提供商的各种大语言模型，为可定制的[智能代理](https://www.librechat.ai/docs/features/agents)添加工具
- 开源且可自托管，具有安全的多用户支持
- 未来路线图包括扩展MCP功能支持

### mcp-agent
[mcp-agent]是一个简单、可组合的框架，用于使用模型上下文协议构建智能代理。

**主要功能：**
- 自动管理MCP服务器连接
- 向大语言模型公开来自多个服务器的工具
- 实现[构建有效智能代理](https://www.anthropic.com/research/building-effective-agents)中定义的每种模式
- 支持工作流暂停/恢复信号，例如等待人类反馈

### Microsoft Copilot Studio
[Microsoft Copilot Studio]是微软推出的强大SaaS平台，专为构建自定义AI应用和智能代理设计，使开发人员能够创建、部署和管理复杂的AI解决方案。

**主要特点：**
- 支持MCP工具集成
- 使用MCP服务器扩展Copilot Studio代理
- 利用微软统一、管控和安全的API管理解决方案

### oterm
[oterm]是一个Ollama终端客户端，允许用户创建聊天和智能代理。

**主要特点：**
 - 支持与Ollama连接的多个可定制聊天会话，具备丰富工具功能
 - 完整支持MCP工具

### Roo Code
[Roo Code](https://roocode.com)通过MCP提供AI编码辅助。

**主要功能：**
- 支持MCP工具和资源
- 与开发工作流集成
- 可扩展的AI能力

### Sourcegraph Cody
[Cody](https://openctx.org/docs/providers/modelcontextprotocol)是Sourcegraph的AI编码助手，通过OpenCTX实现MCP。

**主要功能：**
- 支持MCP资源
- 与Sourcegraph的代码智能集成
- 使用OpenCTX作为抽象层
- 计划未来支持更多MCP功能

### SpinAI
[SpinAI](https://spinai.dev)是一个开源TypeScript框架，用于构建可观察的AI代理。该框架提供原生MCP兼容性，使代理能够与MCP服务器和工具无缝集成。

**主要功能：**
- 为AI代理内置MCP兼容性
- 开源TypeScript框架
- 可观察的代理架构
- 原生支持MCP工具集成

### Superinterface
[Superinterface](https://superinterface.ai)是一个AI基础设施和开发者平台，用于构建支持MCP、交互式组件、客户端函数调用等功能的应用内AI助手。

**主要功能：**
- 在通过React组件或脚本标签嵌入的助手中使用来自MCP服务器的工具
- SSE传输支持
- 使用来自任何AI提供商的任何AI模型（OpenAI、Anthropic、Ollama等）

### TheiaAI/TheiaIDE
[Theia AI](https://eclipsesource.com/blogs/2024/10/07/introducing-theia-ai/)是一个用于构建AI增强工具和IDE的框架。[AI驱动的Theia IDE](https://eclipsesource.com/blogs/2024/10/08/introducting-ai-theia-ide/)是一个建立在Theia AI之上的开放且灵活的开发环境。

**主要功能：**
- **工具集成**：Theia AI使AI代理（包括Theia IDE中的代理）能够利用MCP服务器进行无缝工具交互
- **可定制提示**：Theia IDE允许用户定义和调整提示，动态集成MCP服务器以定制工作流
- **自定义代理**：Theia IDE支持创建利用MCP功能的自定义代理，使用户能够即时设计专用工作流

Theia AI和Theia IDE的MCP集成为用户提供了灵活性，使它们成为探索和适应MCP的强大平台。

**了解更多：**
- [Theia IDE和Theia AI MCP公告](https://eclipsesource.com/blogs/2024/12/19/theia-ide-and-theia-ai-support-mcp/)
- [下载AI驱动的Theia IDE](https://theia-ide.org/)

### Windsurf Editor
[Windsurf Editor](https://codeium.com/windsurf)是一个智能IDE，结合了AI辅助和开发者工作流。它具有创新的AI Flow系统，既能实现协作式AI交互，也能支持独立式AI交互，同时保持开发者对过程的控制。

**主要功能：**
- 革命性的AI Flow范式，用于人类与AI协作
- 智能代码生成和理解
- 丰富的开发工具，支持多模型

### Witsy
[Witsy](https://github.com/nbonamy/witsy)是一个AI桌面助手，支持Anthropic模型和将MCP服务器作为大语言模型工具。

**主要功能：**
- 支持多个MCP服务器
- 工具集成，可执行命令和脚本
- 本地服务器连接，增强隐私和安全性
- 可从Smithery.ai轻松安装
- 开源，适用于macOS、Windows和Linux

### Zed
[Zed](https://zed.dev/docs/assistant/model-context-protocol)是一个具有内置MCP支持的高性能代码编辑器，专注于提示模板和工具集成。

**主要功能：**
- 提示模板在编辑器中以斜杠命令形式呈现
- 工具集成，增强编码工作流
- 与编辑器功能和工作区上下文紧密集成

### OpenSumi
[OpenSumi](https://github.com/opensumi/core)是一个帮助您快速构建AI原生IDE产品的框架。

**主要功能：**

### Daydreams
[Daydreams](https://github.com/daydreamsai/daydreams)是一个用于在链上执行任何操作的生成式代理框架。

**主要功能：**
- 在配置中支持MCP服务器
- 提供MCP客户端

### Apify MCP Tester

[Apify MCP Tester](https://github.com/apify/tester-mcp-client)是一个开源客户端，使用服务器发送事件（Server-Sent Events，SSE）技术连接到任何MCP服务器。
它是一个独立的Apify Actor，专为通过SSE测试MCP服务器而设计，支持授权头（Authorization header）。
它采用原生JavaScript（传统风格）编写并托管在Apify平台上，使您无需任何配置即可运行。

**主要特点：**
- 通过SSE连接到任何MCP服务器
- 与[Apify MCP服务器](https://apify.com/apify/actors-mcp-server)无缝协作，可与一个或多个Apify [Actors](https://apify.com/store)交互
- 根据上下文和用户查询智能调用各种工具（如服务器支持）

## 为您的应用添加MCP支持

如果您已经为您的应用添加了MCP支持，欢迎提交拉取请求（Pull Request）将其添加到此列表。MCP集成可以为您的用户提供强大的上下文感知AI能力，并使您的应用成为不断壮大的MCP生态系统的一部分。

添加MCP支持的优势：
- 让用户能够引入自己的上下文和工具
- 加入快速发展的互操作AI应用生态系统
- 为用户提供灵活的集成选择
- 支持本地优先的AI工作流

想要在您的应用中实现MCP，请查看我们的[Python SDK](https://github.com/modelcontextprotocol/python-sdk)或[TypeScript SDK文档](https://github.com/modelcontextprotocol/typescript-sdk)
